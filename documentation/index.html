LISPBUILDER


 

Abstract

LISPBUILDER, or Common Lisp Application Builder is part of the Common Lisp Gardeners project. LISPBUILDER contains several packages for useful for developing cross-platform applications in Common Lisp. These packages are:

  • LISPBUILDER-CAL3D: 3D character animation
  • LISPBUILDER-CLAWK: AWK for Lisp
  • LISPBUILDER-LEXER: A Lisp lexer
  • LISPBUILDER-ODE: Physics
  • LISPBUILDER-OPENRM: OpenGL Scene Graph
  • LISPBUILDER-REGEX: Regular exression handling
  • LISPBUILDER-SDL: Window and event management, rendering graphics, and sound support.
  • LISPBUILDER-SDL-GFX: 2D graphical effects such as zooming, rotation, drawing circles, polygons and squares
  • LISPBUILDER-SDL-IMAGE: Loading many 2D image formats such as PNG and JPG
  • LISPBUILDER-SDL-MIXER: Multi-channel audio mixing
  • LISPBUILDER-SDL-NET: Simple socket networking
  • LISPBUILDER-SDL-TTF: True-type font rendering
  • LISPBUILDER-WINDOWS: CFFI wrappers for Windows Windows windows.h.
  • LISPBUILDER-YACC: YACC for Lisp

Installation

Windows Installation

  1. Get Lispworks Personal Edition.
  2. Get the Edi Weitz's Lisp Starter Pack.
  3. Download config.lisp and place into the same directory as the Lisp Starter Pack in (2) above.
  4. Run the Lisp Starter Pack from (2), selecting the LISPBUILDER-* packages to install
  5. Start Lispworks. Load (don't Open) the start.lisp file found in the Lisp Starter Pack directory
  6. Installation is now complete. See Using Lispbuilder on how to use a LISPBUILDER package.

Linux Installation

The LISPBUILDER-SDL-* libraries are ASDF-INSTALL compatible. For example the following will install LISPBUILDER-SDL on systems that are configured for ASDF-INSTALL. 

      (asdf-install:install "lispbuilder-sdl-gfx_7.1.tgz")
  • ASDF: For users of SBCL, OpenMCL, ECL or ACL, ASDF is already installed. Otherwise, Linux users please refer to the official ASDF README for installation instructions.
  • CFFI: It's always best to use the latest development tarball for CFFI available here.

Several LISPBUILDER packages rely on external dynamic linked libraries (.so or DLL's). Use your distro's package manager to download and install these libraries.

NOTE: CFFI will attempt to load the .so libraries from the standard Linux .so library search path.

Windows Alternate Installation

This section describes a more detailed process of installing and configuring LISPBUILDER. For example, when the user has an existing Lisp environment installed.

Prerequisites

Some LISPBUILDER packages (LISPBUILDER-SDL) require that the asdf.lisp and CFFI are installed. The installation of these packages is descibed in following section.

Configuring ASDF and CFFI

This section describes the installation of the ASDF and CFFI packages for CLISP in Windows.

  • install CLISP to, e.g. 
    c:\programme\clisp-2.38\
  • create a directory asdf in the clisp-2.38 directory
    • c:\programme\clisp-2.38\asdf\
  • save asdf.lisp in the asdf directory 
    c:\programme\clisp-2.38\asdf\asdf.lisp
  • download CFFI and unpack it (use 7-Zip or another program to unpack .tgz files) in the asdf directory, rename the directory to "cffi". So you should have a file like: 
    c:\programme\clisp-2.38\asdf\cffi\cffi.asd

Package Installation

This section describes the installation of the LISPBUILDER-SDL package.

  • download the latest version of the lispbuilder-sdl package from sourceforge and unpack it to the asdf directory, e.g. 
    c:\programme\clisp-2.38\asdf\lispbuilder-sdl_0.7.1\lispbuilder-sdl.asd

Binary Installation

This section describes the installation of the SDL binaries.

  • install the SDL dynamic library by doing one of the following:
    • download SDL and copy the SDL.dll to following system folder: 
      c:\windows\system32
    • download the lispbuilder-sdl-binaries package from sourceforge and unpack it into the asdf directory, e.g. 
      c:\programme\clisp-2.38\asdf\lispbuilder-sdl\lispbuilder-sdl-binaries.asd
      c:\programme\clisp-2.38\asdf\lispbuilder-sdl\bin\SDL.dll

 

Using LISPBUILDER

Enter the following at the REPL to load the LISPBUILDER-SDL package:

      (asdf:operate 'asdf:load-op :lispbuilder-sdl)

ASDF will take care of loading the CFFI and :lispbuilder-sdl dependencies. The SDL.dll library will also be loaded into the Lisp image at this time.

Replace :lispbulder-sdl above to load the other LISPBUILDER packages. For example:

  • (asdf:operate 'asdf:load-op :lispbuilder-sdl-image) will load the LISPBUILDER-SDL-IMAGE package.
  • (asdf:operate 'asdf:load-op :lispbuilder-windows) will load the LISPBUILDER-WINDOWS package.

Now run one of the examples.


 

Running examples from LISPBUILDER-SDL-EXAMPLES

Enter the following at the REPL to load the examples in the LISPBUILDER-SDL-EXAMPLES package:

      (asdf:operate 'asdf:load-op :lispbuilder-sdl-examples)
